WHAT CAVE DO?

CAVE HIDE YOU ROCK AND SHINY THING FROM BAD PEOPLE. YOU PUT TOKEN IN CAVE, THEY NO SHOW IN YOU WALLET. EVEN IF BAD PEOPLE MAKE YOU SHOW ALL WALLET, THEY NO SEE WHAT IN CAVE.

CAVE USE NFT AS KEY. YOU PICK NFT, YOU HIDE TOKEN WITH NFT. ONLY YOU WITH NFT CAN TAKE TOKEN OUT.

WHY ME NEED CAVE?

PROTECT FROM BAD PEOPLE

BAD PEOPLE WITH WRENCH CAN MAKE YOU SHOW WALLET. BUT THEY NO FIND TOKEN IN CAVE!

PROTECT FROM TOO MUCH RULE PEOPLE

SOMETIMES AUTHORITY PEOPLE WANT SEE ALL YOU HAVE. CAVE HELP YOU KEEP SOME PRIVATE. EVEN WHEN FORCED SHOW EVERYTHING, CAVE ASSETS HIDDEN BEHIND NFT.

HOW CAVE HIDE THINGS?

CAVE NO KEEP SIMPLE LIST LIKE "NFT #1 → 100 TOKEN". TOO EASY FOR SNOOP PEOPLE TO FIND. INSTEAD, CAVE USE SECRET METHOD TO STORE DATA WITH SPECIAL KEY SYSTEM THAT LOOK RANDOM.

YOU WITH RIGHT NFT CAN ACCESS YOU TOKEN FAST AND EASY. BUT SNOOP PEOPLE NEED DO MUCH WORK TO BUILD MAP OF ALL BALANCES. NOT IMPOSSIBLE, JUST HARDER.

IMPORTANT: CAVE NO MAKE YOU INVISIBLE! WHEN YOU PUT TOKEN IN CAVE OR TAKE OUT, TRANSACTION ON PUBLIC LEDGER. PEOPLE CAN SEE YOU DO SOMETHING WITH CAVE. CAVE JUST MAKE IT HARDER TO BUILD COMPLETE MAP OF ALL BALANCES WITHOUT DOING MUCH WORK.

HOW USE CAVE?

THREE CAVE ACTION

IN CAVE: PUT TOKEN IN CAVE

  • YOU PICK NFT FOR USE AS KEY
  • YOU PICK WHAT TOKEN PUT IN (ONE OR MANY)
  • YOU SAY HOW MUCH
  • TOKEN GO IN CAVE, HIDE WITH NFT

OUT CAVE: TAKE TOKEN FROM CAVE

  • YOU SHOW NFT KEY
  • YOU PICK WHAT TOKEN TAKE OUT (ONE OR MANY)
  • YOU SAY HOW MUCH
  • CAVE GIVE YOU TOKEN (MINUS SMALL FEE)

LOOK CAVE: CHECK WHAT IN CAVE

  • YOU SHOW NFT KEY
  • YOU PICK WHAT TOKEN CHECK
  • CAVE TELL YOU HOW MUCH YOU HAVE
  • NO FEE FOR LOOK!

CAVE FEE

WHEN YOU TAKE TOKEN OUT CAVE, CAVE TAKE SMALL FEE. THIS KEEP CAVE RUNNING.

FEE NOW:

0.1% OF WITHDRAWAL (THAT 1 TOKEN PER 1000 TOKEN)

FEE RULES:

  • FEE ONLY ON TAKE OUT, NOT ON PUT IN
  • FEE NEVER MORE THAN 1% (MAX PROTECTION)
  • CURRENT FEE: 0.1%
  • NO FEE FOR LOOK CAVE

FEE EXAMPLE

YOU TAKE OUT 1000 XRD → FEE IS 1 XRD → YOU GET 999 XRD

COMPONENT INTERFACE

FOR DEVELOPER WHO WANT TALK DIRECT TO CAVE:

instantiate_cave()

Returns: (Global<Hypercave>, Bucket)
Purpose: CREATE NEW CAVE + OWNER BADGE

MAKE NEW CAVE COMPONENT. GIVE YOU OWNER BADGE TO CONTROL CAVE.

in_cave(nft_proof, resources)

Parameters:
- nft_proof: Proof (MUST BE EXACTLY 1 NFT)
- resources: Vec<Bucket> (TOKENS TO PUT IN)
Returns: Nothing

PUT TOKEN IN CAVE. USE NFT AS KEY. CAN PUT MANY TOKEN TYPE AT ONCE.

out_cave(nft_proof, withdrawals)

Parameters:
- nft_proof: Proof (MUST BE EXACTLY 1 NFT)
- withdrawals: Vec<(ResourceAddress, Decimal)>
Returns: Vec<Bucket> (YOU TOKEN MINUS FEE)

TAKE TOKEN FROM CAVE. SHOW NFT KEY. SAY WHAT WANT AND HOW MUCH. FEE DEDUCTED AUTOMATIC.

query_balances(nft_proof, resource_addresses)

Parameters:
- nft_proof: Proof (MUST BE EXACTLY 1 NFT)
- resource_addresses: Vec<ResourceAddress>
Returns: Vec<Option<Decimal>>

CHECK WHAT IN CAVE WITHOUT TAKE OUT. SHOW NFT KEY. ASK FOR SPECIFIC TOKEN. GET BALANCE OR NONE IF NOT IN CAVE.

PICK NFT THAT YOU NO CAN LOSE EASY. IF YOU USE NORMAL NFT AND SEND AWAY OR LOSE, YOU TOKEN IN CAVE GONE FOREVER! SOUL-BOUND OR RECALLABLE BEST FOR SAFE!
